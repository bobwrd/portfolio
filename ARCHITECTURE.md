# Architecture

## Repo layout

```
apps/web/            unified Vite+React frontend
worker/               unified Hono Worker entry
  moe/                 Margin of Error + Mini Projects API (mounted at /api)
  onebook/              OneBook API (mounted at /api/onebook)
packages/finance/     pure-TS risk engine, used by worker/onebook and apps/web
content/              Margin of Error's baked content source
scripts/              content-baking and data-build scripts
wrangler.toml         single Worker config: two D1 bindings, one KV binding
```

## Routing

Frontend (`apps/web/src/App.tsx`):

- `/` — landing page
- `/writing`, `/writing/weekly[/:slug]`, `/writing/personal[/:slug]`, `/writing/others[/:slug]`, `/writing/about`, `/writing/why`, `/writing/changed-my-mind`, `/writing/changelog`, `/writing/contact` — Margin of Error
- `/mini`, `/mini/verdict/*`, `/mini/ledger/*`, `/mini/observatory/*`, `/mini/arena/*`, `/mini/lab/*`, `/mini/docket/*` — Mini Projects
- `/onebook`, `/onebook/history`, `/onebook/instruments[/:ticker]`, `/onebook/settings` — OneBook dashboard
- `/ask` — Ask showcase

Worker (`worker/index.ts`):

- `/api/*` — Margin of Error + Mini Projects (content, likes, contact, newsletter, verdict, ledger, observatory, distlab)
- `/api/onebook/*` — OneBook (auth, portfolios, brokers, groups, transactions, preferences, sessions)

## Content pipeline

`scripts/bake-content.mjs` reads `content/articles/*.md`, `content/profile.md`, `content/verdict/verdict_cases.json`, `content/ledger/*`, `content/observatory/observatory.json`, `content/distlab/*` and writes `worker/generated/content.json`, imported directly by the worker (Workers have no filesystem at runtime). Runs automatically as part of `npm run build`.

Verdict cases are generated by an external weekly automation and synced in via `scripts/sync-drive.mjs`; that automation no longer cross-posts into `content/articles/` — Verdict content lives only under `/mini/verdict`.

## Data

Two D1 databases, kept separate:

- Margin of Error DB — `likes`, `contact_submissions`, `newsletter_signups`, `verdict_drafts`.
- OneBook DB — `users`, `login_tokens`, `portfolios`, `positions`, `broker_connections`, `price_cache`, `quote_cache`, `fx_rate_cache`, `groups`, `position_groups`, `news_cache`, `transactions`, `broker_connection_errors`, `user_preferences`, `sessions`.

OneBook also uses a KV namespace for sessions and OAuth state.

## Secrets

Set via `wrangler secret put <NAME>`, never committed:

- `VERDICT_PASSWORD` — gates `/api/verdict/submit`
- `TOKEN_ENCRYPTION_KEY`, `STATE_SIGNING_SECRET` — OneBook auth/broker token encryption
- `ALPACA_API_KEY_ID`, `ALPACA_API_SECRET_KEY`, `ALPHA_VANTAGE_API_KEY`
- `SCHWAB_CLIENT_ID`, `SCHWAB_CLIENT_SECRET`, `TRADIER_CLIENT_ID`, `TRADIER_CLIENT_SECRET`
